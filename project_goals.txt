Data Cleaning Roadmap
Handle Missing Values:

Impute BATH (median) and PROPERTYSQFT (median) - avoid mean due to outliers

Flag records with missing geolocation data for review

Convert BEDS to integer after imputation

Address Outliers:

Cap extreme prices (e.g., $195M penthouse) using IQR method:

python
Q1 = df['PRICE'].quantile(0.25)
Q3 = df['PRICE'].quantile(0.75)
df['PRICE'] = np.where(df['PRICE'] > Q3 + 3*(Q3-Q1), Q3 + 3*(Q3-Q1), df['PRICE'])
Remove properties with <100 sqft or >20k sqft (invalid measurements)

Feature Engineering:

Create Price Zones (target variable):

python
bins = [0, 300000, 750000, 2000000, float('inf')]
labels = ['Low', 'Medium', 'High', 'Very High']
df['PRICE_ZONE'] = pd.cut(df['PRICE'], bins=bins, labels=labels)
Generate new features:

PRICE_PER_SQFT = PRICE / PROPERTYSQFT

BED_BATH_RATIO = BEDS / BATH

DISTANCE_TO_CENTER (compute from coordinates to Manhattan CBD)

Geospatial Processing:

Use SUBLOCALITY as neighborhood indicator

Cluster coordinates into districts using K-Means (5-10 clusters)

Drop redundant columns: BROKERTITLE, MAIN_ADDRESS, ADMINISTRATIVE_AREA_LEVEL_2

Categorical Encoding:

One-hot encode: TYPE (Condo/House), PROPERTY_CATEGORY

Target encode: SUBLOCALITY (high cardinality)

Performance Metric:

Final Advice: Start with XGBoost/Random Forest for baseline, then progress to NN with geospatial features. 
Prioritize feature engineering over model complexity - NYC housing patterns are highly location-dependent. Validate models against gentrification trends (e.g., Brooklyn vs. Queens).
